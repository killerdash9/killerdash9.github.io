# Step 1 Analysis

## Project Snapshot

- 목표: 정적 HTML, CSS, JavaScript만으로 반응형 개인 프로페셔널 웹사이트를 만든다.
- 디자인 방향: Windows XP 계열의 `Clean XP` 스타일, 친숙한 창틀/타이틀바/작업표시줄 느낌, 하지만 과도한 복고 장식은 피한다.
- 게임 방향: 상단 `Games` 메뉴를 두고, 추가 게임 확장 여지를 남긴다.
- 추가 기능 설정: `Tetris`를 포함 대상에 두되, 1단계에서는 구조만 열어두고 우선순위를 낮게 잡는다.

## Recommended Safe First Loop

가장 안전한 첫 루프는 **정보 구조와 정적 골격만 먼저 만드는 것**이다.

- 홈, 소개, 경력, 스킬, Games 섹션의 앵커 구조를 만든다.
- Clean XP 느낌의 창틀, 작업표시줄, 버튼 스타일만 먼저 구현한다.
- 실제 소개 문구는 확인된 항목만 넣고, 비어 있는 항목은 `[사람 확인 필요]`로 둔다.
- 게임은 `Games` 메뉴와 Tetris 컨테이너, 상태 표시만 먼저 둔다.
- Tetris 본 구현은 다음 루프로 미루고, 현재는 메뉴 진입점만 확보한다.

## Key Risks

- CV 기반 정보가 제한적이어서 소개 문구를 과하게 추측하면 안 된다.
- 게임 확장 시 JS 상태 관리가 커질 수 있다.
- XP 느낌을 내면서도 과도한 장식으로 무거워지지 않도록 조절해야 한다.
- 모바일 터치 조작은 데스크톱 키보드와 같은 규칙으로 매핑해야 혼란이 적다.

## Notes On Safety

- 토큰 파일명만 기록하고, 값은 읽거나 저장하지 않는다.
- 연락처, 주소 등은 웹사이트에 기본 노출하지 않는 방향이 안전하다.
- 확정되지 않은 프로필 내용은 반드시 `[사람 확인 필요]`로 남긴다.
